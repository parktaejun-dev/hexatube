# BalanceTube 개발 목표 및 유저 시나리오

## 📋 프로젝트 개요

**BalanceTube**는 사용자의 YouTube 시청 기록을 분석하여 콘텐츠 소비의 균형을 시각화하고, 편향된 콘텐츠 소비를 개선하기 위한 맞춤형 추천을 제공하는 Android 애플리케이션입니다.

### 핵심 가치
- **콘텐츠 소비의 균형**: 다양한 분야의 콘텐츠를 골고루 접하도록 유도
- **데이터 기반 인사이트**: 객관적인 데이터로 사용자의 시청 패턴 파악
- **프라이버시 우선**: 모든 데이터는 로컬에 저장되며 외부 서버로 전송되지 않음

---

## 🎯 개발 목표

### 1. 핵심 목표

#### 1.1 균형잡힌 콘텐츠 소비 촉진
- 사용자가 자신의 시청 패턴을 인지하고 편향된 콘텐츠 소비를 개선할 수 있도록 지원
- 6개 주요 카테고리(지식, 엔터테인먼트, 라이프스타일, 예술/음악, 자기계발, 소셜/크리에이터)에 걸쳐 균형잡힌 시청 유도

#### 1.2 직관적인 시각화 제공
- 레이더 차트를 통한 카테고리별 소비 비율의 직관적 시각화
- 기간별(최근 7일, 30일, 전체) 분석 제공으로 시간에 따른 변화 추적

#### 1.3 개인화된 추천 시스템
- 사용자가 상대적으로 덜 소비한 카테고리의 콘텐츠를 우선 추천
- 초보자 친화적인 콘텐츠(2-14분 길이) 중심 추천으로 진입 장벽 완화

#### 1.4 프라이버시 중심 설계
- 모든 데이터 처리는 로컬에서 수행
- 이메일 주소는 SHA-256 해시로 저장
- 언제든지 완전한 데이터 삭제 가능

### 2. 세부 목표

#### 기능적 목표
- ✅ Google OAuth를 통한 안전한 인증
- ✅ YouTube Data API v3를 활용한 시청 기록 동기화
- ✅ 규칙 기반 콘텐츠 분류 시스템 (향후 ML 기반으로 개선 예정)
- ✅ 실시간 데이터 분석 및 시각화
- ✅ 카테고리별 맞춤형 추천

#### 기술적 목표
- Clean Architecture 및 MVVM 패턴 적용으로 유지보수성 향상
- Jetpack Compose를 활용한 선언적 UI 구현
- Kotlin Coroutines 및 Flow를 통한 비동기 처리
- Room 데이터베이스를 통한 효율적인 로컬 데이터 관리
- Hilt를 통한 의존성 주입으로 코드 결합도 감소

#### 사용자 경험 목표
- 3단계 이내의 간단한 온보딩 프로세스
- 5초 이내의 시청 기록 동기화 (최대 200개 비디오)
- 직관적이고 깔끔한 Material Design 3 UI
- 오프라인에서도 이전 분석 결과 확인 가능

## 🔍 문제 정의 및 해결 방안

### 문제 1: 필터 버블과 편향된 콘텐츠 소비
**현상**: YouTube 알고리즘은 사용자가 선호하는 콘텐츠를 지속적으로 추천하여 콘텐츠 소비가 특정 분야에 편중되는 경향이 있습니다.

**영향**:
- 새로운 분야에 대한 관심 저하
- 정보 편식으로 인한 시야 협소화
- 다양한 지식과 경험의 기회 상실

**해결 방안**:
- 6개 카테고리에 걸친 시청 기록 분석
- 레이더 차트를 통한 시각적 불균형 인지
- 상대적으로 적게 소비한 카테고리의 콘텐츠 추천

### 문제 2: 시청 패턴에 대한 인식 부족
**현상**: 대부분의 사용자는 자신이 어떤 유형의 콘텐츠를 주로 소비하는지 정확히 인지하지 못합니다.

**영향**:
- 무의식적인 시간 낭비
- 자기계발이나 학습 콘텐츠 소비 부족
- 의도하지 않은 콘텐츠 소비 패턴 고착화

**해결 방안**:
- 데이터 기반의 객관적인 시청 패턴 분석
- 기간별 비교를 통한 변화 추적
- 카테고리별 시청 시간 및 비율 제공

### 문제 3: 새로운 분야 탐색의 어려움
**현상**: 관심 없는 분야의 콘텐츠를 찾고 시작하는 것이 심리적으로 어렵습니다.

**영향**:
- 새로운 분야 진입 장벽
- 지식과 경험의 다양성 부족
- 잠재적 관심사 발견 기회 상실

**해결 방안**:
- 2-14분 길이의 초보자 친화적 콘텐츠 우선 추천
- 카테고리별 맞춤 검색 키워드 활용
- 최신성과 인기도를 고려한 추천 알고리즘

## 👥 유저 페르소나 및 시나리오

### 페르소나 1: 대학생 민준 (22세)
**특징**:
- 게임과 엔터테인먼트 콘텐츠를 주로 시청
- 학업과 자기계발에 관심은 있지만 실행력 부족
- 스마트폰 사용 시간이 하루 4-5시간

**니즈**:
- 재미있게 시청하면서도 생산적인 콘텐츠 소비
- 시험 기간에는 학습 관련 콘텐츠 발견
- 자신의 시청 패턴 파악 및 개선

**사용 시나리오**:
1. **인식 단계**
   - BalanceTube를 처음 실행하고 Google 계정으로 로그인
   - "시청 기록 동기화" 버튼을 눌러 최근 200개 영상 분석
   - 레이더 차트에서 엔터테인먼트가 70%, 지식이 5%임을 확인하고 놀람

2. **탐색 단계**
   - "균형 회복 추천" 버튼을 눌러 지식 카테고리 추천 확인
   - "과학 설명 영상", "TED 강연" 등 짧고 흥미로운 영상 5개 발견
   - 관심 가는 5분짜리 과학 영상 하나를 YouTube에서 시청

3. **개선 단계**
   - 일주일 후 다시 앱을 실행하여 변화 확인
   - 지식 카테고리가 5%에서 15%로 증가한 것을 확인
   - 자기계발 카테고리도 탐색하기로 결심

### 페르소나 2: 직장인 수진 (29세)
**특징**:
- 출퇴근 시간에 YouTube 시청 (주로 Vlog, 라이프스타일)
- 자기계발과 커리어 성장에 관심이 많음
- 효율적인 시간 활용을 중요시함

**니즈**:
- 여가 시간의 생산적 활용
- 새로운 관심사와 취미 발견
- 균형잡힌 콘텐츠 소비로 시야 확대

**사용 시나리오**:
1. **현황 파악**
   - 앱 설치 후 최근 30일 시청 기록 분석
   - 라이프스타일 55%, 소셜/크리에이터 30%, 나머지 15%로 불균형 확인
   - 예술/음악 카테고리가 거의 없음을 인식

2. **새로운 발견**
   - 예술/음악 추천 리스트 확인
   - "클래식 음악 입문", "미술관 투어" 등의 영상 발견
   - 출퇴근 시간에 클래식 음악 감상 시작

3. **습관 형성**
   - 매주 일요일 앱에서 균형 상태 확인
   - 가장 낮은 카테고리의 추천 영상 1-2개 시청을 루틴화
   - 한 달 후 더 균형잡힌 레이더 차트 확인하며 성취감 느낌

### 페르소나 3: 중학생 지우 (15세)
**특징**:
- YouTube로 게임, 밈, 챌린지 영상 주로 시청
- 진로와 학업에 대한 고민 시작
- 부모님이 의미있는 콘텐츠 시청을 권장함

**니즈**:
- 재미와 학습의 균형
- 다양한 분야 탐색을 통한 진로 탐색
- 자율적인 콘텐츠 소비 관리

**사용 시나리오**:
1. **부모님의 권유로 시작**
   - 부모님과 함께 앱 설치 및 로그인
   - 시청 기록 분석 결과, 엔터테인먼트 85%, 지식 3% 확인
   - 부모님과 함께 목표 설정: 지식 콘텐츠 20% 달성

2. **재미있는 학습 발견**
   - 지식 카테고리 추천에서 "쿠르츠게작트" 애니메이션 과학 영상 발견
   - 짧고 재미있는 형식에 흥미를 느껴 여러 편 시청
   - 역사, 우주, 기술 등 다양한 주제에 관심 생김

3. **자발적 사용**
   - 일주일에 한 번씩 스스로 앱을 열어 균형 확인
   - 새로운 카테고리를 하나씩 시도해보며 관심사 확대
   - 학교 과제에도 참고할 수 있는 유익한 영상 발견

## 📊 상세 사용 시나리오

### 시나리오 A: 첫 사용 (온보딩)
1. **로그인** (LoginScreen)
   - "Google로 로그인" 버튼 터치
   - Google 계정 선택 및 권한 승인
   - YouTube Data API 접근 권한 부여

2. **메인 화면 진입** (HomeScreen)
   - 초기 상태: 빈 레이더 차트와 "시청 기록 동기화" 버튼 표시
   - 안내 메시지: "YouTube 시청 기록을 분석하여 균형을 확인하세요"

3. **시청 기록 동기화**
   - "시청 기록 동기화" 버튼 터치
   - 로딩 인디케이터 표시 (2-5초)
   - 최근 200개 비디오 자동 분류 및 저장
   - 완료 후 레이더 차트 자동 생성

4. **결과 확인**
   - 6개 카테고리별 시청 비율 레이더 차트로 표시
   - 가장 많이/적게 소비한 카테고리 하이라이트
   - 기간 선택 (최근 7일/30일/전체)

### 시나리오 B: 추천 콘텐츠 탐색
1. **추천 화면 진입**
   - 홈 화면에서 "균형 회복 추천" 버튼 터치
   - 또는 하단 네비게이션에서 "추천" 탭 선택

2. **추천 로딩**
   - 가장 낮은 카테고리 자동 감지
   - YouTube API를 통한 관련 영상 검색
   - 2-14분 길이의 초보자 친화적 콘텐츠 필터링
   - 최신 5개 영상 목록 표시

3. **콘텐츠 선택**
   - 영상 썸네일, 제목, 채널명, 길이 확인
   - 관심 있는 영상 터치
   - YouTube 앱에서 해당 영상 재생

4. **재분석**
   - YouTube에서 시청 완료 후 BalanceTube로 복귀
   - "시청 기록 동기화"로 새로운 시청 기록 반영
   - 업데이트된 레이더 차트 확인

### 시나리오 C: 정기적인 모니터링
1. **주간 체크인**
   - 매주 일요일, 앱 실행
   - "시청 기록 동기화"로 최신 데이터 반영
   - 기간을 "최근 7일"로 설정하여 이번 주 분석

2. **변화 확인**
   - 지난주 대비 카테고리별 변화 인식
   - 목표로 했던 카테고리의 증가 확인
   - 새로운 최저 카테고리 파악

3. **다음 주 계획**
   - 새로운 최저 카테고리의 추천 영상 확인
   - 관심 가는 영상 2-3개 선택
   - 일주일간 해당 영상들 시청 계획

### 시나리오 D: 데이터 관리
1. **설정 화면 진입**
   - 하단 네비게이션에서 "설정" 탭 선택
   - 사용자 이메일(해시), 저장된 비디오 수, 시청 이벤트 수 확인

2. **데이터 삭제**
   - "모든 데이터 삭제" 버튼 터치
   - 확인 다이얼로그: "정말 모든 로컬 데이터를 삭제하시겠습니까?"
   - "확인" 선택 시 즉시 모든 데이터 삭제
   - 홈 화면으로 이동, 초기 상태로 복귀

3. **로그아웃**
   - "로그아웃" 버튼 터치
   - 로그인 화면으로 복귀
   - 로컬 데이터는 보존 (원할 경우 다시 로그인하여 확인 가능)

## 📝 결론

BalanceTube는 단순한 YouTube 분석 도구를 넘어, 사용자가 더 의도적이고 균형잡힌 콘텐츠 소비 습관을 형성하도록 돕는 디지털 웰빙 솔루션입니다.

프라이버시를 최우선으로 하면서도 강력한 분석과 추천 기능을 제공함으로써, 사용자들이 필터 버블에서 벗어나 더 넓은 세상을 탐험할 수 있도록 지원합니다.

이 프로젝트는 MVP(Minimum Viable Product) 단계이며, 사용자 피드백을 바탕으로 지속적으로 개선하고 발전시켜 나갈 예정입니다.
